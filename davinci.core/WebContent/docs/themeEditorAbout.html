About the Theme Editor

A "theme" is a collection of CSS files that provide a consistent set of visual styling rules that work across a widget library (i.e., Dojo). A theme typically defines the following characteristics across a variety of widgets:

  • Border styling (thickness, color, rounding of box corners)
  • Background styling (color, background images, background repeats)
  • Text styling
  • Margins and padding
  • Icons used by the widgets (arrows, "x" close icons, +/- icons for trees)

The Dojo toolkit ships with a handful of standard themes (tundra, soria, nihilo, noir), and with Dojo 1.5, there is a new theme, Claro.

Maqetta ships with Claro as the default theme, which can be customized using the Claro theme editor. With the theme editor, you can customize the default styling for various widgets. For example, you might want buttons to have a dark blue border with light brown background.

Launching the Theme Editor

To launch the theme editor, select Open > Theme Editor from the Maqetta menu bar. Your browser window should look something like this:

All of the supported widgets are displayed on the theme editor canvas. The widgets are divided into three sections: Global Styling, Primitive Widgets and Container Widgets. Click on any widget to select it. When a widget is selected, its properties can be customized in the Properties palette.

Global Styling

The Global Styling section is for setting styles that apply globally. There are two types of elements that can be set globally: text and widgets.

The Generic text block is for setting text styles that apply to all widgets that contain text elements. To see what widgets are affected, select the generic text block and experiment with setting various text styles in the Text and Font section of the Properties palette. HTML elements that contain text (such as <label>, <span>, <h1>-<h3> and <a>) are also affected by the styles set on the generic text block.

The Global Background and Border widget represents a generic widget and allows you to globally set the background and border for all widgets. These properties are under the Background and Border sections of the Properties palette.

Note: Changes made with the Global Background and Border widget will override changes previously made to individual widgets. If you want to customize individual widgets for your theme, you should make all global changes first, then make the individual widget changes.

Customizing the look of a particular type of widget

To customize the visual styling for a particular type of widget, select the widget on the theme editor canvas and modify the styling properties for that widget in the Properties palette.

Some complex widgets, such as Calendar, allow different styling for different "subwidgets", which are particular sub-elements to the given widget. When you select a complex widget, a panel opens with a selectable list of the subwidgets contained in the widget. By default the outer container widget is selected. If you select one of the subwidgets, that subwidget is outlined in red on the widget. Any changes made in the Properties palette will affect only the component that is selected.

For Calendar, you can customize the styling on several things, including the year, the month, the days of the week (Sun to Sat), and the dates (0 to 31).

Widget States

The term "widget states" refers to the various interactive states that a widget might have. For example, the Dojo Button widget has three widget states: Normal, Hover and Disabled. When the mouse hovers over a button or a button is inactive/disabled, Dojo changes certain CSS properties, such as border color, font color, and background properties, in order to provide visual feedback at runtime.

In the theme editor you can style each widget state using the States palette, which contains a list of all the states. You click on a state in the State palette to make that state the current state. As you click on each state, you will notice that the widget styles change. You can change any widget's default style for a given state by selecting the state, then selecting the widget and changing its properties using the Properties palette. You can also use the Global styling widgets to style widgets globally for any of the widget states.

For complext widgets, some of the subwidgets can be styled in the Hover and Selected states, as well as the Normal state. Note that for the other states, many of the subwidgets are grayed out when the state is selected, because styling has no effect on the subwidgets in those states.
