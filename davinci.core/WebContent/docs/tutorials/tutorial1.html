Introducing the Page Editor

Creating a new HTML page in the page editor
  1. Click New > HTML File... on the Maqetta menubar.
  2. The Create New HTML File dialog will be displayed with a default filename.
  3. Click the Create button to create a new file with the default filename.
  4. A new blank HTML file is opened in the page editor, with the filename displayed at the top.

Flow layout versus absolute layout

Maqetta defaults to flow layout. With flow layout, in most cases the page content begins at the top/left and then flows left-to-right and then top-to-bottom. This is the "natural" layout approach used by HTML and CSS, whose original designs were centered on flowable documents. If you start adding simple UI controls such as buttons and textboxes to the page, you'll see that the first widget snaps to the top/left of the page and the second widget will be placed either just to the right of the first widget or just below the first widget.

Maqetta also supports absolute layout. With absolute layout, widgets and HTML elements are placed at particular (x,y) locations on the page. Other sections of the documentation provide more information about how to switch into absolute layout mode.

Add Dojo controls to the page
  1. In the Widgets palette, click on the Dojo Controls section to open that section.
  2. Drag a Text Box widget onto the canvas.
  3. Drag a Button widget onto the HTML page canvas to add a button to the page. Notice that Maqetta automatically pops up the widget's data entry popup so that you can immediately define the text for the button's label.
  4. Enter some text (e.g., "Press me") and hit Return. You will see a button with the label you provided.
  5. Double-click on the Button widget. Notice that Maqetta re-opens the data entry popup.
  6. Change the text to say "Click-me" and hit Return.
  7. Drag a Calendar widget onto the canvas.
  8. The Calendar widget is initially set with today's date.
  9. To change the Calendar's date, double-click on the Calendar widget and edit the date.
  10. In the Widgets palette, click on the Dojo Controls section title to close the section.

Maqetta automatically prompts the user for initial data for a number of widgets, such as Button, Calendar, <label>, <a> and <h1> - <h3>. Later you can double-click on the widget to edit the initial data.

Add HTML elements to the page
  1. In the Widgets palette, click on the HTML section to open that section.
  2. Drag a <div> widget onto the canvas.
  3. Drag a <label> widget onto the <div> on the canvas. This will put the <label> inside the <div>.
  4. Enter some text for the <label> and hit Return.
  5. Click on the HTML section title to close the section.

Add an image to the page
  1. The Files palette is in the lower-left section of the Maqetta workspace.
  2. You can expand the palette by placing the mouse on the upper edge of the blue divider and dragging up with the mouse.
  3. From the Files palette, drag and drop the "SampleBanner.jpg" file onto the page canvas.

To add your own images, right-click (ctrl-click on the Mac) on the Files palette to bring up the context menu, and select Upload File. Uploaded files are displayed in the Files palette where you can drag and drop them onto the canvas.

Working with widgets

To work with a widget, you must first select it. You can then size it, move it, style it or set its properties.

Selecting a widget

You can select a widget in two ways:

  • Click on a widget on the page canvas.
  • Click on a widget's entry in the Outline palette.

A gray border is displayed around a widget to show that it is selected.

You can select multiple widgets by holding down the Ctrl key (Command key on Mac) as you select the widgets.

Resizing, moving and styling a widget

  • To change the size of a widget, select it and pull one of the resize handles (little white boxes along the corners and sides of the selection border).

Some widgets such as Button do not have resize handles because their size depends on their interior content.

  • In flow layout, you can move widgets in two ways:
    • On the canvas, select the widget, then place the mouse on the selection border and then drag it to a different location.
    • On the Outline palette, drag the widget to a new location within the Outline palette. (The Outline palette allows widgets to be moved into any position in the widget hierarchy.)
  • In absolute layout, you can also drag widgets to different locations. However, the effect or rearranging absolutely positioned widgets within the Outline palette is more complex:
    • Widgets may or not not move to different physical locations on the canvas if they are moved to a different location in the Outline palette.
    • Moving widgets in the Outline palette will alter drawing order, and therefore might cause one widget to draw behind or in front of other widgets.

Nesting widgets

  • You can nest widgets by dragging and dropping a widget onto another widget (the target must be a container widget).
  • Nested widgets can be moved into a different container widget, or outside of any widgets to become a top-level widget on the page.

In certain scenarios, nesting widgets might affect how some CSS properties are inherited, and therefore might affect the visual styling of a particular widget.

Styling widgets

  • To change the look of a widget, click on the widget to select it and then change its values using the Properties palette.

Deleting widgets
  1. Select one of the widgets in the Page Editor canvas.
  2. Delete the widget in one of the following ways:
    • Click on the delete icon on the page editor toolbar.
    • Hit the Delete key on your keyboard (for Windows only - currently not working on the Mac).
    • Bring up the widget's context menu (right-click on Windows / ctrl-click on Mac) and choose delete.
  • You can also delete a widget from the Outline palette:
    1. Select the widget in the Outline palette
    2. Bring up the widget's context menu (right-click on Windows / ctrl-click on Mac) and choose delete.

Saving and opening files

Click on the Save icon on the page editor toolbar to save the current HTML file.

Saving with Save as

Use Save as to save the file under a different file name:

  1. Click on the Save as icon on the page editor toolbar.
  2. A dialog window will pop up with the directory list from your user workspace folder (which is also displayed in the Files palette).
  3. File name: - enter a new file name or select an existing file from the list (using an existng filename will overwrite the existing file).
  4. Parent folder: - select any folder from the list (the root user workspace folder is indicated by a "."). Do not use the themes folder since this is reserved for Maqetta theme files.
  5. The file will be saved with the new file name, and will also remain open on the canvas with the new file name.

Opening Files

There are two ways to open an existing file:

  1. Find the file in the Files palette and double-click on it.
  2. Select Open > Open File... from the Maqetta menu bar. A dialog window will pop up allowing you to select the file to open.
