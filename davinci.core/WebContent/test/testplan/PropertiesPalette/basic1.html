Properties palette basic tests (part 1)

Common properties

Steps

  1. Standard setup things: Clear the browser cache. Launch the Maqetta server. Login if necessary. In some cases, you'll want to a virgin user environment by creating a whole new user login.
  2. Issue a "New HTML File" command.
  3. Make the Properties palette visible.
  4. Drag a Button onto the canvas.
  5. Under Common properties, type in values for id, class and title.
  6. Click on split view icon. Verify that source code shows the values for the above properties.
  7. Save, close, reopen.
  8. Click on split view icon. Verify that source code shows the values for the above properties.

Expected results

  1. In source pane, find the markup for the Button. you should see the values you supplied as the attribute values for 'id', 'class' and 'title'.

Widget-specific

Steps

  1. Standard setup things: Clear the browser cache. Launch the Maqetta server. Login if necessary. In some cases, you'll want to a virgin user environment by creating a whole new user login.
  2. Issue a "New HTML File" command.
  3. Make the Properties palette visible.
  4. Drag a Button, a TextBox and an HTML DIV onto the canvas.
  5. Select the Button, then the TextBox, then the DIV. The widget-specific property list should change based on the widget.
  6. Type in values for the various widget-specific properties.
  7. Click on split view icon. Verify that source code shows the values for the above properties.
  8. Save, close, reopen.
  9. Select the Button, then the TextBox, then the DIV.
  10. Click on split view icon. Verify that source code shows the values for the above properties.

Expected results

  1. As you change widget-specific properties, the new values should be reflected in the visual appearance of the widget on the canvas.
  2. Upon re-opening the file, the widget-specific properties that you entered should show in the Properties palette as you select each widget.
  3. The widget-specific properties should appear in the HTML markup for each widget.

Events properties

Steps

  1. Standard setup things: Clear the browser cache. Launch the Maqetta server. Login if necessary. In some cases, you'll want to a virgin user environment by creating a whole new user login.
  2. Issue a "New HTML File" command.
  3. In Firefox, open Firebug and show the console. In Chrome, open up the JavaScript console.
  4. Make the Properties palette visible.
  5. Drag two TextBoxes onto the canvas.
  6. For one of the TextBoxes, type in "console.log('something or other')" for each of the click, mouse and focus/blur event fields.
  7. Click on the "Save" icon on the main toolbar to save the new file. The asterisks that indicates that the file is "dirty" should go away.
  8. Click on the "X" close button for the editor tab for this file
  9. In the Files palette, double-click on the file you just created to re-open the file.
  10. Click Preview in Browser on the toolbar.
  11. In the browser preview, move the mouse over and out of the TextBox, and change the focus between the TextBoxes.

Expected results

  1. You should see the console.log() messages you put into the event fields. (Note: for Chrome, you have to make sure that the developer tools window is working against the preview in browser tab instead of Maqetta itself.)
