Properties palette basic tests (part 3)

Backgrounds, borders, margins, padding properties using shortcuts

Steps

  1. Create a virgin user environment navigating away from daVinci, clearing browser cache, killing the server, removing users/ folder, restarting the server, and navigating browser back to daVinci. (For server-hosted, create a new user)
  2. Issue a "New HTML File" command.
  3. Drag a ContentPane onto the canvas, and then put a LABEL, TextBox and Button inside the ContentPane.
  4. Go to Properties palette and for all widgets, change all of the background properties, the border shortcut property, the border-radius shortcut property, the margin shortcut property and paddding shortcut property to new values, making sure to test each property at least once for at least one widget.
    Note that you will get a prompt in some cases warning you that you might be changing a global CSS style sheet. Say OK to these warnings, but note that this will changes your theme CSS files. After running this test, you'll want to make sure you recreate your user so that the theme CSS files are restored to their original state.
    Note that background-color and background-image can collide. An opaque background-image will mask the background color.
    The background image dialog shows a gradient option. No need to verify that feature for the basic test plan, especially since most browsers don't support that particular CSS3 feature yet.
  5. Click on the "Save" icon on the main toolbar to save the new file. The asterisks that indicates that the file is "dirty" should go away.
  6. Click on the "X" close button for the editor tab for this file
  7. In the Files palette, double-click on the file you just created to re-open the file.

Expected results

  1. As you change the properties, the canvas should update so that the widgets reflect the styling changes you made.
  2. As you change the properties, the Properties palette's text entry and preview areas should update to reflect the changes you made.
  3. After save and reopen, the saved file should open and its contents should be the same as how it was when you clicked on the Save button.

Border, margin, padding detail properties

Steps

  1. Create a virgin user environment navigating away from daVinci, clearing browser cache, killing the server, removing users/ folder, restarting the server, and navigating browser back to daVinci. (For server-hosted, create a new user)
  2. Issue a "New HTML File" command.
  3. Drag two DIV elements onto the canvas.
  4. Select the second DIV. Double-click and set text to "abc".
  5. Go to Properties palette and set the border shortcut to "1px solid black", border-radius shortcut to "4px", 'margin' shortcut to "10px" and 'padding' shortcut to "5px".
  6. Under 'border' property on Properties palette, choose the option to show "props". Set border-width to 3px, border=style to dotted, border-color to red.
    You should see these changes on the design canvas.
  7. For each of the above properties, issue "remove value".
    DIV should be restored to its default settings.
  8. Using Undo, restore original properties: border shortcut to 1px solid black, border-radius shortcut to 4px, margin shortcut to 10px, padding shortcut to 5px.
  9. Choose the option to show border sides. Set left border to "3px dotted green".
    You should see these changes on the design canvas.
  10. Choose the option for borders to show all. Set border-top-width to "7px" and border-right-color to blue.
    You should see these changes on the design canvas.
  11. Under 'border-radius' property on Properties palette, choose the option to show details. Set border-top-right-radius to 20px.
    You should see these changes on the design canvas.
  12. Under margins and padding, turn on "show t/r/b/l". Set margin-left to 20px and padding-top to 10px.
    You should see these changes on the design canvas.
  13. Click on the "Save" icon on the main toolbar to save the new file. The asterisks that indicates that the file is "dirty" should go away.
  14. Click on the "X" close button for the editor tab for this file
  15. In the Files palette, double-click on the file you just created to re-open the file.

Expected results

  1. As you change the properties, the canvas should update so that the widgets reflect the styling changes you made.
  2. As you change the properties, the Properties palette's text entry and preview areas should update to reflect the changes you made.
  3. After save and reopen, the saved file should open and its contents should be the same as how it was when you clicked on the Save button.

Fonts and Text properties

Steps

  1. Standard setup things: Clear the browser cache. Launch the daVinci server. Login if necessary. In some cases, you'll want to setup a virgin user environment by creating a whole new user login.
  2. Issue a "New HTML File" command.
  3. Drag a DIV onto the canvas.
  4. Drag a Label inside the DIV and enter "Label 1" as the label text.
  5. Drag another Label inside the DIV and enter "Label 2" as the label text.
  6. Click in the white space inside the DIV (outside of the labels) so the DIV is selected.
  7. Go to Properties palette and change various text-related values in the Text twisty: font-family, font-size, font-color, bold, italic, text-decoration, text-align.
    "Label 1" and "Label 2" should inherit the values set on the DIV.
    The labels should move to left, center and right as you click on the DIV's text-align buttons.
  8. Click inside "Label 2" to select it.
  9. Go to Properties palette and change some of the text-related values in the Text twisty. Note how the values set on "Label 2" override the inherited values from the DIV.
  10. Drag a TextBox and a DIV onto the canvas.
  11. Select the TextBox. Double-click to bring up text input editing control and set the value to "abc"
  12. Select the DIV. In the Properties palette, set 'width' to '200px', double-click on DIV, and set the value to "abc def abc def abc def abc def abc def".
  13. Go to Properties palette and for both the TextBox and DIV, change various text-related values in the Text twisty. For the DIV, change all of the values, particularly white-space:nowrap and white-space:pre (in both cases, the text should extend past the end of the DIV). For TextBox, you might find that some of the values have no effect (e.g., line-height and text-indent).
    Click OK when prompt asks OK to change global style sheet. Note that you should recreate a new user after running this test so that in the future you will be using original CSS theme files.
  14. Click on the "Save" icon on the main toolbar to save the new file. The asterisks that indicates that the file is "dirty" should go away.
  15. Click on the "X" close button for the editor tab for this file
  16. In the Files palette, double-click on the file you just created to re-open the file.

Expected results

  1. As you change the properties, the canvas should update so that the widgets reflect the styling changes you made.
  2. As you change the properties, the Properties palette's text entry and preview areas should update to reflect the changes you made.
    The font-family and font-size areas should show the new values as text.
    The 'color' widgets should be solid filled with the chosen text color.
    The bold, italic, underline and justification icons should be highlighted to match the toggling state for those properties.
  3. The text for the labels inside the DIV should show the values you set for the DIV.
    The text for "Label 2" should show a mixture of the inherited DIV values and the values set on the label.
    Any values set on the label element should override the values set on the DIV element.
  4. As you click the text-align buttons on the DIV, you should see the labels move within the DIV (left-justify, center, right-justify).
  5. After save and reopen, the saved file should open and its contents should be the same as how it was when you clicked on the Save button.
