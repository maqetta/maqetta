Properties palette basic tests (part 4)

CSS cascade features

Steps

  1. Create a virgin user environment navigating away from Maqetta, clearing browser cache, killing the server, removing users/ folder, restarting the server, and navigating browser back to Maqetta. (For server-hosted, create a new user)
  2. Clone the "claro" theme into a new theme using the "New Theme" command. For example, create theme "mytheme".
  3. Drag a Button onto canvas.
  4. Issue a "New HTML File" command.
  5. Drag a Button onto canvas.
  6. Go to Properties palette and go to 'width' property. Click on the right-pointing icon at the right side of the 'width' property to show the CSS cascade.
    You should see a list of entries, element.style and then various CSS selectors, at least one of which should include within it ".claro .dijitButtonNode".
    Up above in the type-in area of the 'width' property, type "200px", then press Enter to try to commit the value.
    This operation if completed would modify one of the CSS rules in the (read-only) Claro theme. Any rules that would modify a read-only theme should be disabled. This isn't working completely yet. Here is a pending bug: https://github.com/maqetta/maqetta/issues/258
    If you see the "about to change a global CSS rule, are you sure", click "Cancel".

  7. Click on the Switch themes icon on the toolbar, and change to your new theme ("mytheme").
  8. Go again to Properties palette and go to 'width' property and the CSS cascade view for that property. Up above in the type-in area of the 'width' property, type "200px", then press Enter to try to commit the value. You should see an "about to change a global CSS rule, are you sure".
    This operation attempts to change one of the style rules within one of the CSS files for your new "mytheme" theme.
    Click "OK" to make a global change such that all Buttons will have width 200px. The currently selected button should now be 200px wide.

  9. Drag another Button onto the canvas. It should be 200px wide also.
  10. Click on the Switch themes icon on the toolbar, and change back to ("claro").
    The buttons should revert back to their default size.
  11. Click on the Switch themes icon on the toolbar, and change back to ("mytheme").
    The buttons should be 200px wide again.
  12. Click on the "Save" icon on the main toolbar to save the new file. The asterisks that indicates that the file is "dirty" should go away.
  13. Click on the "X" close button for the editor tab for this file
  14. In the Files palette, double-click on the file you just created to re-open the file.

Expected results

  1. See text above for expected results.

app.css files

Steps

  1. Create a virgin user environment navigating away from Maqetta, clearing browser cache, killing the server, removing users/ folder, restarting the server, and navigating browser back to Maqetta. (For server-hosted, create a new user)
  2. Double-click on the app.css file in the Files palette.
  3. Add the following lines to the bottom of the file: 
    #myapp.claro .class1.dijitButton .dijitButtonNode {
	width:200px;
}
#label1 {
	font-size:30px;
}
  4. Save and close the file.
  5. Issue a "New HTML File" command.
  6. Drag two Buttons and two LABELs onto canvas.
  7. Select the second Button, go to Properties palette and set class to 'class1'.
  8. Select the second LABEL, go to Properties palette and set id to 'label1'.
  9. Click on the "Save" icon on the main toolbar to save the new file. The asterisks that indicates that the file is "dirty" should go away.
  10. Click on the "X" close button for the editor tab for this file
  11. In the Files palette, double-click on the file you just created to re-open the file.

Expected results

  1. Second button should have width of 200px and second LABEL should have font size of 30px.
