User management basic tests

Login page

Steps

  1. Clear your browser cache.
  2. Launch the Maqetta server such that the server requires a user login (i.e., do not run server with localInstall=true option).
  3. Browse to the Maqetta root URL without the user name (e.g., http://localhost:8080/maqetta, not http://localhost:8080/maqetta/JohnSmith).

Expected results

  1. You should see the login page, with two parts:
    • "Please sign in", followed by Login and Password entry boxes
    • New users - please sign up

New user sign up

Steps

  1. Clear your browser cache.
  2. With the login page showing (see previous section), click on the please sign up hyperlink
  3. You should now see four fields for new user signup: Login name, email, password and confirm password. Enter valid values for these fields and click the button to create a new user.

Expected results

  1. The Maqetta application should appear with its various palettes at left, right and bottom and a toolbar across the top.
  2. The right side of the toolbar should have your login name followed by a logout hyperlink.

Log out (continuation of previous testing steps)

Steps

  1. Click the logout link

Expected results

  1. The login screen should reappear.

Existing user sign in

Steps

  1. Clear your browser cache.
  2. If you are current logged into the Maqetta server as a user, click the logout hyperlink at the top right of the toolbar. Else, launch the Maqetta server with user management activated (i.e., do not run server with localInstall=true option) and browse to the Maqetta root URL without the user name (e.g., http://localhost:8080/maqetta, not http://localhost:8080/maqetta/JohnSmith).
  3. Type in the user name and password for a user account that created previously and click on the Login button.

Expected results

  1. The Maqetta application should appear with its various palettes at left, right and bottom and a toolbar across the top.
  2. The right side of the toolbar should have your login name and, further to the right, by a logout hyperlink.

Local install, single-version of Maqetta (localInstall=true)

This section verifies that there are no user login if the server is running on localhost with the "-Dmaqetta.localInstall=true" option.

Steps

  1. Clear your browser cache.
  2. Launch the Maqetta server on localhost with option "-Dmaqetta.localInstall=true".
  3. Browse to the Maqetta root URL without the user name (e.g., http://localhost:8080/maqetta, not http://localhost:8080/maqetta/JohnSmith).

Expected results

  1. No login screen. Instead, you should immediately jump to to Maqetta application, with its various palettes at left, right and bottom and a toolbar across the top.
  2. The right side of the toolbar should say "localhost user".
