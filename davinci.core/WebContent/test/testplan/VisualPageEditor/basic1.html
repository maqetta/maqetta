Visual page editor basic tests (part 1 - editor toolbar)

New HTML file, preview, save, reopen, preview, close

Steps

  1. Standard setup things: Clear the browser cache. Launch the daVinci server. Login if necessary. In some cases, you'll want to setup a virgin user environment by creating a whole new user login.
  2. From the New menu on the banner, choose New HTML File...
  3. Click OK to accept the default file name (file1.html)
  4. Drag/drop any widget onto canvas
  5. Click on the "Preview in browser" icon on editor toolbar to open another browser tab which shows the current HTML file running directly in browser
  6. View the new browser tab, then close it
  7. Back in daVinci, click the "Save" button on editor toolbar
  8. Close the editor by clicking "x" on editor's tab
  9. Double-click on the filename in Files palette to reopen
  10. Click on the "Preview in browser" icon on editor toolbar
  11. View the new browser tab, then close it
  12. Back in daVinci, close the editor by clicking "x" on editor's tab

Expected results

  1. After each Preview in Browser, a new browser tab (or window) should open and the same thing you see in the drawing canvas within daVinci should appear in the new browser tab.

SaveAs

Steps

  1. Standard setup things: Clear the browser cache. Launch the daVinci server. Login if necessary. In some cases, you'll want to a virgin user environment by creating a whole new user login.
  2. On main menubar, choose "New HTML File" to create a new file
  3. Drag a few widgets onto the canvas (e.g., Button, TextBox, Calendar).
  4. Click on the "SaveAs" icon on the main toolbar to save the new file. In the dialog that appears, choose a new file name.
  5. Click on the "X" close button for the editor tab for the newly renamed file
  6. In the Files palette, double-click on the file you just created

Expected results

  1. After completing the SaveAs operation, the old editor (with the old filename) should be replaced by a new editor (with the new filename). Its contents should be the same as how it was when you clicked on the SaveAs button.
  2. Upon reopening, the saved file should open and its contents should be the same as how it was.

Cut/copy/paste/delete

Steps

  1. Standard setup things: Clear the browser cache. Launch the daVinci server. Login if necessary. In some cases, you'll want to a virgin user environment by creating a whole new user login.
  2. On main menubar, choose "New HTML File" to create a new file
  3. Drag a few widgets onto the canvas (e.g., Button, TextBox, Calendar).
  4. Select one of the widgets. Click on Cut. Then click on Paste. The cursor should change to a crosshair cursor. Click somewhere on canvas and the widget that was cut from the document should now be pasted back in.
  5. Select one of the widgets. Click on Copy. Then click on Paste. The cursor should change to a crosshair cursor. Click somewhere on canvas and the widget that was copied from the document should now be pasted back in.
  6. Select one of the widgets. Click on Copy. Select a different widget. Click on Delete. Then click on Paste. The cursor should change to a crosshair cursor. Click somewhere on canvas and the first widget (the copied widget) should now be pasted back in, not the deleted widget.

Expected results

  1. The Cut command should have removed the widget, and Paste added it back in.
  2. The Copy command should not remove the widget. Paste should add a second copy.
  3. The Delete command should have removed the widget, and Paste should not have added it back in.

Undo/Redo

Steps

  1. Standard setup things: Clear the browser cache. Launch the daVinci server. Login if necessary. In some cases, you'll want to a virgin user environment by creating a whole new user login.
  2. On main menubar, choose "New HTML File" to create a new file
  3. Drag a few widgets onto the canvas (e.g., Button, TextBox, Calendar).
  4. Click Undo until all widgets disappear.
  5. Click Redo until all widgets reappear.
  6. Select a widget and delete it.
  7. Click Undo to restore the widget.
  8. Click Redo remove that widget again.
  9. Select one of the widgets. Set the 'id' property to something (e.g., "abc") in the Properties palette.
  10. Click Undo to remove the property setting.
  11. Click Redo to restore the property setting.
  12. Click the Save button, then close the file.
  13. In Files palette, reopen the file. After viewing the file, close the file.

Expected results

  1. Operations should undo/redo correctly.
  2. Upon reopening, the file should reflect the state of the document when it was last saved.

Switch layout: Flow vs Absolute

Steps

  1. Standard setup things: Clear the browser cache. Launch the daVinci server. Login if necessary. In some cases, you'll want to a virgin user environment by creating a whole new user login.
  2. On main menubar, choose "New HTML File" to create a new file
  3. Drag TextBox, ComboBox, and Button onto canvas. They should be added in "flow" layout (left-to-right, top-to-bottom) because that's the default preference value.
  4. Click on switch layout icon. A dialog should appear. Choose Absolute.
  5. Drag a Calendar onto canvas, dropping into middle of page. Calendar should be positioned absolutely (i.e., position:absolute).
  6. Save the file and close it.
  7. Double-click on Sample1.html. Drag/drop a Button into middle of page. Because Sample1.html was last in Flow Layout, the Button should be inserted into the doc in flow layout mode.
  8. Reopen the file you just created.
  9. Drag a MultiSelect onto canvas, dropping into middle of page. MultiSelect should be positioned absolutely (i.e., position:absolute). The last choice for layout state should be persisted into file.

  1. (See instructions above for expected results.)

Then click away from the rich text editor to cause your changes to be added to the document as a yellow sticky note that is absolutely positioned.

  6. Sticky note should be right where it was before.
