For desktop devices, Maqetta offers four different composition types:

  • Desktop hi-fi page - This option creates a desktop user interface using a final-form, production-ready CSS theme to render the widgets, using flow layout. With flow layout, in most cases the page content begins at the top/left and then flows left-to-right and then top-to-bottom. This is the "natural" layout approach used by HTML and CSS, whose original designs were centered on flowable documents. It is usually better for your developers if you create a user interface using flow layout instead of absolute layout because flow layout is usually more adaptable to different browser window sizes and different user font size choices.
  • Desktop hi-fi sketch - This option creates a WYSIWYG desktop user interface using a final-form, production-ready CSS theme to render the widgets, but instead of flow layout, Maqetta initializes the page to use absolute layout where widgets are positioned at absolute locations on the page, similar to how drawing tools work. Absolute layout is often better for quick UI mockups.
  • Desktop lo-fi page - Same as Desktop hi-fi page in that it defaults to flow layout, but uses a low fidelity CSS theme that renders widgets in black-and-white using a CSS theme that provides a pseudo hand-drawn appearance. Low fidelity mockups are often good for early-stage UI mockups where you want colleagues to focus on the overall design and flow of your UI rather than focusing on the visual styling such as colors and fonts.
  • Desktop lo-fi sketch - Uses a low fidelity CSS theme that provides a pseudo hand-drawn appearance, and initializes the page to use absolute layout where widgets are positioned at absolute locations on the page, similar to how drawing tools work. This option is often a good choice for early-stage UI mockups.

The other composition types are for mobile devices. When you choose one of these composition types, Maqetta will open the page editor within an actual size replica of the selected device. With the current release of Maqetta, mobile composition types always use a high fidelity CSS theme and always initialize with absolute layout.

Creating a new HTML page in the page editor (desktop hi-fi page)
  1. Bring up the New > HTML File... dialog if it isn't showing already. The New HTML File dialog will be displayed with a default filename.
  2. Set Composition Type to "Desktop hi-fi page". (The first option.)
  3. Click the Create button to create a new file with the default filename.
  4. A new blank HTML file is opened in the page editor, with the filename displayed at the top. Your workspace should now look something like this:

    Click on the image to enlarge it.
  5. In the Widgets palette, click on the Dojo Controls section to open that section.
  6. Find the Text Box widget in the list of Dojo Controls. The controls are alphabetized, so the TextBox entry will be towards the bottom of the list of controls. Drag a TextBox widget onto the canvas.
  7. Find the Button widget in the list of Dojo Controls. Drag/drop 2 Button widgets onto the canvas.

    Note that the TextBox and Buttons have final-form visual styling (using the "claro" theme from Dojo) and are using flow layout.
  8. In the Widgets palette, click on the Dojo Controls section title to close the section.
  9. Click the "x" button on the editor tab just under the toolbar to close this page editor. Click OK for the prompt that asks "are you sure?"

Using the page editor
  1. Bring up the New > HTML File... dialog. The New HTML File dialog will be displayed with a default filename.
  2. Set Composition Type to "Desktop hi-fi page". (The first option.)
  3. Click the Create button to create a new file with the default filename.
  4. A new blank HTML file is opened in the page editor
  5. In the Widgets palette, click on the Dojo Controls section to open that section.
  6. Find the Text Box widget in the list of Dojo Controls. The controls are alphabetized, so the TextBox entry will be towards the bottom of the list of controls. Drag a TextBox widget onto the canvas.
  7. Click on the Undo icon on the toolbar. This will undo the previous operation, causing the TextBox widget to disappear. Now click on the Redo icon . The TextBox widget will re-appear.
    Maqetta supports multiple undo. Operations can be undone back to the last Open or Save operation.
  8. Drag a Button widget onto the HTML page canvas to add a button to the page. Notice that Maqetta automatically pops up the widget's data entry popup so that you can immediately define the text for the button's label.

    Maqetta automatically prompts the user for initial data for a number of widgets, such as Button, ComboBox, DataGrid, Tree, <label>, <a> and <h1> - <h3>.

  9. Enter some text (e.g., "Press me") and hit Return. You will see a button with the label you provided.
  10. Double-click on the Button widget. Notice that Maqetta re-opens the data entry popup.
  11. Change the text to say "Click-me" and hit Return.
  12. Drag a Calendar, Color Palette and ComboBox widget onto the canvas. Note that ComboBox pop ups a widget entry popup just like Button did. Click OK to accept the default data for ComboBox.
    Note that some widgets are added to the right of the previous widget and other widgets are added under the previous widget. This is because some widgets default as "block-level" widgets, which means they, in effect, also establish their own separate block (or "paragraph"), whereas other widgets are designed as "inline" widgets, which means they are placed inline within the current block (or "paragraph") and behave from a layout perspective similar to typed text or small inline images. Maqetta simply honors the default layout behavior for each widget. If you need to change between block and inline for a particular widget, you can go to the Properties palette, Layout section, and change the value of the 'display' property. If you want an inline widget to be placed on a separate line below the previous widget, you can also use the context menu and issue a "Surround with DIV" command, which places the inline widget inside of the DIV element (and DIV elements defaults to block-level).
  13. In the Widgets palette, click on the Dojo Controls section title to close the section.

Add HTML elements to the page
  1. In the Widgets palette, click on the HTML section to open that section.
  2. Drag a <div> widget onto the canvas.
  3. Drag a <label> widget onto the <div> on the canvas. This will put the <label> inside the <div>.
  4. Enter some text for the <label> and hit Return.
  5. Click on the HTML section title to close the section.

Add an image to the page
  1. The Files palette is in the lower-left section of the Maqetta workspace.
  2. You can expand the palette by placing the mouse on the upper edge of the blue divider and dragging up with the mouse.
  3. From the Files palette, drag and drop the "SampleBanner.jpg" file onto the page canvas.

To add your own images, right-click (ctrl-click on the Mac) on the Files palette to bring up the context menu, and select Upload File. Uploaded files are displayed in the Files palette where you can drag and drop them onto the canvas or reference them from 'background-image' properties.

Working with widgets

To work with a widget, you must first select it. You can then size it, move it, style it or set its properties.

Selecting a widget

You can select a widget in three ways:

  • Click on a widget on the page canvas.
  • Click on a widget's entry in the Outline palette.
  • In split view (where both the design canvas and HTML source are visible at the same time), you can often select part or all of the source code corresponding to a particular widget, and often the given widget will show as selected on the page canvas.

A gray border is displayed around a widget to show that it is selected.

You can select multiple widgets by holding down the Ctrl key (Command key on Mac) as you select the widgets.

Resizing, moving and styling a widget

  • To change the size of a widget, select it and pull one of the resize handles (small white boxes along the corners and sides of the selection border).

    Some widgets such as Button do not have resize handles because their size depends on their interior content.

  • In flow layout, you can move widgets in two ways:
    • On the canvas, select the widget, then place the mouse on the selection border and then drag it to a different location within the existing "flow" of widgets.
    • On the Outline palette, drag the widget to a new location within the Outline palette. (The Outline palette allows widgets to be moved into any position in the widget hierarchy.)
  • In absolute layout, you can also drag widgets to different locations. However, the effect of rearranging absolutely positioned widgets within the Outline palette is more complex:
    • Widgets may or not not move to different physical locations on the canvas if they are moved to a different location in the Outline palette.
    • Moving widgets in the Outline palette will alter drawing order, and therefore might cause one widget to draw behind or in front of other widgets.

Sometimes layout rules with Maqetta can be complex because fundamentally Maqetta is a WYSIWYG authoring tool for HTML5 and CSS3. Maqetta creates HTML and CSS source files which are ultimately passed to the browser for layout and rendering. Because the layout rules for HTML and CSS are sometimes complex, if you reorder widgets in the Outline palette, hand-edit source code and/or change property values on widgets in the Properties palette, then you are altering the HTML and CSS source files that are sent to the browser, which might trigger various complex CSS layout rules and thereby produce visual results different than what you expected. Maqetta tutorials attempt to demonstrate authoring techniques that minimize chances of confusing results.

Nesting widgets

  • You can nest widgets by dragging and dropping a widget onto another widget (the target must be a container widget).
  • Nested widgets can be moved into a different container widget, or outside of any widgets to become a top-level widget on the page.

In certain scenarios, nesting widgets might affect how some CSS properties are inherited, and therefore might affect the visual styling of a particular widget.

Styling widgets

  • To change the look of a widget, click on the widget to select it and then change its values using the Properties palette.

Deleting widgets
  1. Select one of the widgets in the Page Editor canvas.
  2. Delete the widget in one of the following ways:
    • Click on the delete icon on the page editor toolbar.
    • Hit the Delete key on your keyboard (for Windows only - currently not working on the Mac).
    • Bring up the widget's context menu (right-click on Windows / ctrl-click on Mac) and choose delete.
  • You can also delete a widget from the Outline palette:
    1. Select the widget in the Outline palette
    2. Bring up the widget's context menu (right-click on Windows / ctrl-click on Mac) and choose delete.

Saving and opening files

Click on the Save icon on the page editor toolbar to save the current HTML file.

Saving with Save as

Use Save as to save the file under a different file name:

  1. Click on the Save as icon on the page editor toolbar.
  2. A dialog window will pop up with the directory list from your user workspace folder (which is also displayed in the Files palette).
  3. File name: - enter a new file name or select an existing file from the list (using an existng filename will overwrite the existing file).
  4. Where: - the current folder into which the file can be saved. To the right of the Where dropdown menu is a right-pointing arrow that, when clicked, shows a list of files in your workspace where you can select any folder from the workspace or create a New Folder. Do not use the themes folder since this is reserved for Maqetta theme files.
  5. The file will be saved with the new file name, and will also remain open on the canvas with the new file name.

Opening Files

There are two ways to open an existing file:

  1. Find the file in the Files palette and double-click on it.
  2. Select Open > Open File... from the Maqetta menu bar. A dialog window will pop up allowing you to select the file to open.
