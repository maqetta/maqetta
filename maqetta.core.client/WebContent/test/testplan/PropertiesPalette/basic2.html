Properties palette basic tests (part 2)

Layout properties

Steps

  1. Standard setup things: Clear the browser cache. Launch the Maqetta server. Login if necessary. In some cases, you'll want to setup a virgin user environment by creating a whole new user login.
  2. Issue a "New HTML File" command.
  3. Drag a Button, TextBox and Calendar onto the canvas.
  4. Make the Properties palette visible.
  5. Select the TextBox. Double-click and set value to "abc".
  6. Select the Button. There should be no resize handles. Try to change the Width property on the Properties palette. You should get a prompt telling you that you cannot change a property that is defined on a read-only CSS theme.
  7. Select the TextBox. There should be only horizontal resize handles. Try to change the Width property on the Properties palette. It should work.
  8. Select the Calendar. There should be resize handles for both Width and Height and the Width and Height properties should be enabled on the Properties palette. Change both the width and the height on the Properties palette.
  9. Select the Calendar. The Position property should have no value. Try to move the widget on the canvas. It should snap back to its original position.
  10. Select the Calendar. Change the Position property to "absolute". Try to move the widget on the canvas. It should now move. Change the left and top properties on the Properties palette.
  11. Click on the "Save" icon on the main toolbar to save the new file. The asterisks that indicates that the file is "dirty" should go away.
  12. Click on the "X" close button for the editor tab for this file
  13. In the Files palette, double-click on the file you just created to re-open the file.

Expected results

  1. As you change the properties, the canvas should update so that the widgets reflect the styling changes you made.
  2. After save and reopen, the saved file should open and its contents should be the same as how it was when you clicked on the Save button.

Absolute vs relative positioning

Steps

  1. Standard setup things: Clear the browser cache. Launch the Maqetta server. Login if necessary. In some cases, you'll want to a virgin user environment by creating a whole new user login.
  2. Issue a "New HTML File" command.
  3. On editor toolbar, click on the flow/absolute layout icon, and turn Flow Layout to true.
  4. Make the Properties palette visible.
  5. Drag two ContentPane's onto canvas.
  6. Drag a Button into first ContentPane and a TextBox into second ContentPane.
  7. Select each of the ContentPanes, set position to "absolute" in Properties palette, and drag each ContentPane to a new absolute location.
  8. Select the Button and set position to "absolute" and drag around to a different absolute location.
  9. Restore all elements to position:static (normal CSS flow layout).
  10. Save, reopen.

Expected results

  1. You should be able to change the 'position' property on various types of widgets.

display, opacity, float, clear, overflow

Steps

  1. Standard setup things: Clear the browser cache. Launch the Maqetta server. Login if necessary. In some cases, you'll want to a virgin user environment by creating a whole new user login.
  2. Issue a "New HTML File" command.
  3. Make the Properties palette visible.
  4. Drag a Button, LABEL and DIV onto canvas.
  5. Select LABEL.
  6. Set display to block. The LABEL should now be a block-level element and shift down to the row under the Button.
  7. Select Button.
  8. Set opacity to 0.5. The button should now be semi-transparent.
  9. Set float to right. The button should shift to the right, and LABEL should move to top/left.
  10. Select LABEL.
  11. Set clear to both. The LABEL should shift down to the row under the Button.
  12. Select DIV.
  13. Double-click the DIV and set its text content to "abc def ghi abc def ghi abc def ghi".
  14. Set width to 100px, set white-space to 'nowrap', and set overflow to hidden. The text should be clipped at the edge of the DIV.
  15. Save, close, reopen.

Expected results

  1. Upon reopening, the display, opacity, float, clear and overflow properties should still be in effect.

box-shadow

Steps

  1. Note: box-shadow might not be working yet in 0.6.
  2. Standard setup things: Clear the browser cache. Launch the Maqetta server. Login if necessary. In some cases, you'll want to a virgin user environment by creating a whole new user login.
  3. Issue a "New HTML File" command.
  4. Drag a DIV onto canvas. Double-click the DIV and set its text content to "abc". Set width to "3em".

  6. Save, close, reopen.

Expected results

  1. Upon reopening, the box-shadow property should still be in effect.

z-index

Steps

  1. Standard setup things: Clear the browser cache. Launch the Maqetta server. Login if necessary. In some cases, you'll want to a virgin user environment by creating a whole new user login.
  2. Issue a "New HTML File" command.
  3. Make the Properties palette visible.
  4. Click on the flow layout icon on the editor toolbar and switch to absolute layout.
  5. Drag a Button and TextBox onto canvas such that they overlap. (Notice that each is given a default z-index of 900.)
  6. Select Button. Enter a value of 901 as the value for z-index. The Button should now be above the TextBox.
  7. Select TextBox. Click on "+" icon for z-index to set z-index to 901. The TextBox should now be above the Button.
  8. Select Button. Click on + icon once for z-index to change the z-index to 902. The Button should now be above the TextBox.
  9. Save, close, reopen.

Expected results

  1. Upon reopening, the z-index properties should still be in effect.
